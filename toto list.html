🎯 My recommendation: Build a Personal Profile Page

It's simple, personal, satisfying to see finished, and touches every core concept you need.

📄 What it will include

  1. Header — your name + a short tagline ("Student & future developer 🚀")
  2. Profile picture — a placeholder image or your photo
  3. About section — a paragraph about yourself
  4. Skills list — Python, HTML, CSS (as a bulleted or styled list)
  5. Favorite things — hobbies, games, music (fun section!)
  6. Contact / Social links — even if fake for now (email, GitHub, etc.)
  7. Footer — "Made with ❤️ by [Your Name]"

🧠 What you'll learn building it

🪜 Suggested Building Order

  1. Day 1 — Just the HTML, no styling at all (structure first!)
  2. Day 2 — Add basic CSS: colors, fonts, spacing
  3. Day 3 — Make it centered and nicely laid out (intro to Flexbox)
  4. Day 4 — Add hover effects on links/buttons (your first interactivity!)
  5. Day 5 — Final polish — shadows, rounded corners, responsive tweaks

💡 Why this project is perfect for beginners

  • Small enough to finish in under a week
  • Personal, so you actually care about how it looks
  • Touches 90% of beginner HTML/CSS concepts
  • Easy to keep improving later (add more sections, animations, etc.)

Want me to give you a starter HTML template to begin right now, with empty sections for you to fill in and style yourself?

That's the wireframe — a single-column stacked layout, which is exactly how you'll build it in HTML (top to bottom) before any CSS positioning comes in. Mapping it to actual tags as you code:
une donnée (colonne) dans une ligne d'un tableau
  • Header block → <header> containing an <h1> (name) and <h2> or <p> (tagline)
  • Profile picture → <img> inside that header
  • About → a <section> with a heading + <p>
  • Skills → a <section> with a <ul> of <li> items (the pill badges)
  • Favorite things → a <section> with a few <div>s or another small list
  • Social links → a <nav> or <div> with a few <a> tags
  • Footer → <footer> with your closing line
Build it in this order: write all the HTML first with placeholder text and no styling at all, check that it makes sense top-to-bottom in the browser, then start adding CSS — background colors, spacing, rounding the corners, centering the card on the page. That mirrors the wireframe's flow nicely.
